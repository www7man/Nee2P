01 — Главное 01 — The main thing

Сервер физически не видит ваши сообщения The server physically cannot see your messages

Самая распространённая угроза — «сервис читает вашу переписку». В Nee2P. сервер не может этого сделать, даже если захочет или будет взломан. Ключ шифрования не покидает ваш браузер. The most common threat is "the service reads your messages". In Nee2P. the server cannot do that — even if it wanted to, or if it was breached. The encryption key never leaves your browser.

Что происходит при подключении What happens during the handshake

  • Ваша фраза → через Argon2id → часть ключа (не покидает браузер) Your phrase → through Argon2id → part of the key (never leaves the browser)
  • Браузер генерирует одноразовую X25519-пару (новую при каждом подключении) The browser generates a one-time X25519 key pair (fresh on every connection)
  • Браузеры обмениваются публичными ключами через сервер — он видит только их The browsers exchange public keys through the server — it only ever sees those
  • ECDH + ML-KEM-768 + phrase → итоговый AES-256-GCM ключ (только в браузере) ECDH + ML-KEM-768 + phrase → the final AES-256-GCM key (browser only)
  • Сообщение шифруется: AES-256-GCM(key, iv, plaintext) Each message is encrypted: AES-256-GCM(key, iv, plaintext)
  • Сервер получает {iv, ct} и пересылает второму участнику — содержимое ему недоступно The server receives {iv, ct} and forwards it to the other participant — the contents are not available to it

Что сервер реально видит What the server actually sees 

{
  "iv": "<12 случайных байт, base64>",
  "ct": "<зашифрованный текст, base64>"
}
{
  "iv": "<12 random bytes, base64>",
  "ct": "<encrypted text, base64>"
}

Это всё. Без ключа — бессмысленный набор байт. That's all. Without the key, it's a meaningless pile of bytes.

Как проверить самому How to verify it yourself

Открой DevTools → Network → WS (или вкладку /r/send). Все исходящие пакеты содержат только iv и ct. Plaintext там нет. Open DevTools → Network → WS (or the /r/send tab). Every outgoing packet contains only iv and ct. No plaintext.

03 — Safety Fingerprint

Проверка человеком, что между вами нет посредника A human check that there is no middleman between you

После установки соединения оба участника видят одни и те же 12 слов BIP-39. Это отпечаток ваших публичных ключей — SHA-256 от их конкатенации, отсортированной по значению. Once the connection is established, both participants see the same 12 BIP-39 words. This is the fingerprint of your public keys — SHA-256 over their concatenation, sorted by value.

  • Слова совпали (вы сверили их через голос или другой канал) → между вами нет MITM. The words match (you verified them by voice or another channel) → there is no MITM between you.
  • Слова не совпали → кто-то подменяет ключи. Сессию пересоздать. The words do not match → someone is swapping the keys. Recreate the session.

Где смотреть: в чате → «Безопасность сессии» → 12 слов и полный hex SHA-256. Where to find it: in the chat → "Session security" → 12 words and the full SHA-256 hex.

04 — Время жизни данных 04 — Data lifetime

Только в оперативной памяти, без диска RAM only — never on disk

  • Сообщения хранятся только в RAM сервера, пока сессия активна Messages are stored only in server RAM, for as long as the session is active
  • TTL комнаты — на выбор: 1 час, 6 часов, 24 часа, 3 дня, 7 дней Room TTL is your choice: 1 hour, 6 hours, 24 hours, 3 days, 7 days
  • По истечении TTL (или когда все закрыли вкладку) — комната удаляется, данные исчезают When the TTL expires (or everyone closes the tab) — the room is removed, the data disappears
  • Никакой базы данных, никакого диска, никаких бэкапов No database, no disk, no backups
  • Файлы и голосовые: лежат в RAM как зашифрованный blob (до 5 МБ), удаляются вместе с комнатой Files and voice notes: sit in RAM as encrypted blobs (up to 5 MB), removed together with the room
  • Журнала сообщений на сервере не ведётся The server keeps no message log

Как проверить: исходный код server.js — class Room использует this.history = []. Обычный массив в памяти, без вызовов в БД. How to verify: source code server.js — class Room uses this.history = []. A plain in-memory array, no database calls.

Вы видите код на GitHub, но не можете автоматически проверить, что именно он крутится на нашем сервере. Решение: запустите свой relay (см. выше) или сверьте JS-файлы в DevTools с тем, что в репозитории. You can see the code on GitHub, but you cannot automatically verify that exactly that code is running on our server. Solution: run your own relay (see above), or diff the JS files in DevTools against what's in the repo.

Память сервера после краша Server memory after a crash

Мы говорим, что не логируем — но извне это не подтвердить. Доверяйте RAM-only архитектуре или запустите свой сервер на машине, к которой имеете доступ. We say we do not log — but that cannot be confirmed from outside. Trust the RAM-only architecture, or run your own server on a machine you control.

Компрометация браузера или ОС Compromise of your browser or OS

Если у вас установлено вредоносное расширение, кейлоггер, или ОС под чужим контролем — никакая криптография на стороне приложения не поможет. Это общий предел сквозного шифрования. If you have a malicious extension installed, a keylogger, or an OS under someone else's control — no application-side cryptography will save you. This is a fundamental limit of end-to-end encryption.

09 — Короткий FAQ 09 — Short FAQ

Вы видите IP-адреса пользователей? Do you see user IP addresses?

Технически — да, как любой сервер. IP не связан с содержимым переписки, но если нужна анонимность сетевого уровня — используйте Tor. Technically yes, as any server does. The IP is not tied to message content, but if you need network-level anonymity — use Tor.

Фраза улетает на сервер? Does the phrase leave the device?

Нет. Фраза обрабатывается только в вашем браузере через Argon2id и становится частью ключа шифрования. На сервер уходит только идентификатор комнаты — md5(phrase), по которому невозможно восстановить исходную фразу. No. The phrase is processed only in your browser via Argon2id and becomes part of the encryption key. Only the room identifier reaches the server — md5(phrase), from which the original phrase cannot be recovered.

Что если сервер упадёт во время переписки? What if the server crashes mid-conversation?

Сообщения теряются — они только в RAM. Это намеренно: никакого постоянного хранилища нет. Messages are lost — they only live in RAM. This is intentional: there is no persistent storage.

Можно ли это прослушать на уровне сети? Can this be wiretapped at the network level?

Трафик идёт через HTTPS (TLS). Даже если перехватить его — видны только зашифрованные пакеты {iv, ct}, без AES-ключа их не расшифровать. Traffic goes over HTTPS (TLS). Even if intercepted, only the encrypted {iv, ct} packets are visible — without the AES key they cannot be decrypted.
