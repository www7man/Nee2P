Как убедиться, что мы не врём

Мы не просим доверять нам на слово. Вот что можно проверить самостоятельно.

01 — Главное

Сервер физически не видит ваши сообщения

Самая распространённая угроза — «сервис читает вашу переписку». В Nee2P. сервер не может этого сделать, даже если захочет или будет взломан. Ключ шифрования не покидает ваш браузер.

Что происходит при подключении

  • Ваша фраза → через Argon2id → часть ключа (не покидает браузер)
  • Браузер генерирует одноразовую X25519-пару (новую при каждом подключении)
  • Браузеры обмениваются публичными ключами через сервер — он видит только их
  • ECDH + ML-KEM-768 + phrase → итоговый AES-256-GCM ключ (только в браузере)
  • Сообщение шифруется: AES-256-GCM(key, iv, plaintext)
  • Сервер получает {iv, ct} и пересылает второму участнику — содержимое ему недоступно

Что сервер реально видит

{
  "iv": "<12 случайных байт, base64>",
  "ct": "<зашифрованный текст, base64>"
}

Это всё. Без ключа — бессмысленный набор байт.

Как проверить самому

Открой DevTools → Network → WS (или вкладку /r/send). Все исходящие пакеты содержат только iv и ct. Plaintext там нет.

03 — Safety Fingerprint

Проверка человеком, что между вами нет посредника

После установки соединения оба участника видят одни и те же 12 слов BIP-39. Это отпечаток ваших публичных ключей — SHA-256 от их конкатенации, отсортированной по значению.

  • Слова совпали (вы сверили их через голос или другой канал) → между вами нет MITM.
  • Слова не совпали → кто-то подменяет ключи. Сессию пересоздать.

Где смотреть: в чате → «Безопасность сессии» → 12 слов и полный hex SHA-256.

04 — Время жизни данных

Только в оперативной памяти, без диска

  • Сообщения хранятся только в RAM сервера, пока сессия активна
  • TTL комнаты — на выбор: 1 час, 6 часов, 24 часа, 3 дня, 7 дней
  • По истечении TTL (или когда все закрыли вкладку) — комната удаляется, данные исчезают
  • Никакой базы данных, никакого диска, никаких бэкапов
  • Файлы и голосовые: лежат в RAM как зашифрованный blob (до 5 МБ), удаляются вместе с комнатой
  • Журнала сообщений на сервере не ведётся

Как проверить: исходный код server.js — class Room использует this.history = []. Обычный массив в памяти, без вызовов в БД.

05 — Никаких трекеров и CDN

Браузер ходит только на ваш relay

  • Google Analytics — нет
  • Яндекс.Метрика — нет
  • Facebook Pixel и прочие — нет
  • Google Fonts — убраны (для Tor-совместимости)
  • Внешние CDN — нет. Все библиотеки лежат локально в vendor/

Что входит в vendor

  • react@18.3.1 + react-dom@18.3.1 — UI
  • @babel/standalone@7.29.0 — компиляция JSX в браузере
  • argon2-browser@1.18.0 — Argon2id (WebAssembly)
  • noble-ed25519 — X25519 для браузеров без нативной поддержки
  • mlkem.bundle.mjs — ML-KEM-768 (FIPS 203)
  • bip39-en.js — 2048 английских слов для fingerprint
  • qrcode.min.js — генерация QR-кода (офлайн)

Как проверить: DevTools → Network. Никаких запросов на fonts.googleapis.com, cdn.jsdelivr.net, аналитику и т.п. — только ваш relay.

07 — Запустить локально

Свой relay за минуту

# склонировать репозиторий
git clone https://github.com/TODO/Nee2P
cd Nee2P

# установить зависимости
npm install

# запустить локальный relay
node server.js
# → http://127.0.0.1:8787

Подняв свой relay и открыв DevTools, можно убедиться, что сервер действительно видит только {iv, ct} — и больше ничего.

08 — Честно: чего проверить нельзя

Границы того, что вы можете подтвердить сами

Соответствие prod-кода исходнику

Вы видите код на GitHub, но не можете автоматически проверить, что именно он крутится на нашем сервере. Решение: запустите свой relay (см. выше) или сверьте JS-файлы в DevTools с тем, что в репозитории.

Память сервера после краша

Мы говорим, что не логируем — но извне это не подтвердить. Доверяйте RAM-only архитектуре или запустите свой сервер на машине, к которой имеете доступ.

Компрометация браузера или ОС

Если у вас установлено вредоносное расширение, кейлоггер, или ОС под чужим контролем — никакая криптография на стороне приложения не поможет. Это общий предел сквозного шифрования.

09 — Короткий FAQ

Вы видите IP-адреса пользователей?

Технически — да, как любой сервер. IP не связан с содержимым переписки, но если нужна анонимность сетевого уровня — используйте Tor.

Фраза улетает на сервер?

Нет. Фраза обрабатывается только в вашем браузере через Argon2id и становится частью ключа шифрования. На сервер уходит только идентификатор комнаты — md5(phrase), по которому невозможно восстановить исходную фразу.

Что если сервер упадёт во время переписки?

Сообщения теряются — они только в RAM. Это намеренно: никакого постоянного хранилища нет.

Можно ли это прослушать на уровне сети?

Трафик идёт через HTTPS (TLS). Даже если перехватить его — видны только зашифрованные пакеты {iv, ct}, без AES-ключа их не расшифровать.
