v0.9.15 — Работа на любом reverse-proxy без CDN + картинки в чатеv0.9.15 — Runs behind any plain reverse-proxy (no CDN) + chat images

v0.9.15 июнь 2026June 2026 фиксfixes развёртываниеdeployment вложенияattachments

Stock-сборка теперь работает на любом обычном reverse-proxy (Caddy/nginx) без CDN спереди, и в чате снова грузятся картинки The stock build now works behind any plain reverse-proxy (Caddy/nginx) with no CDN in front, and chat images load again

Критические фиксы развёртыванияCritical deployment fixes

  • server.js: добавлены .mjs и .wasm в MIME-карту. Раньше ES-модули из vendor/ (mlkem.bundle.mjs, noble-ed25519.bundle.mjs) отдавались как application/octet-stream, и браузеры отклоняли их динамический import() — ломалась загрузка ML-KEM-768 на любом деплое без CDN, переписывавшего Content-Type server.js: added .mjs and .wasm to the MIME map. ES modules in vendor/ (mlkem.bundle.mjs, noble-ed25519.bundle.mjs) were served as application/octet-stream, so browsers rejected their dynamic import() — breaking ML-KEM-768 loading on any deploy without a CDN that rewrote Content-Type
  • http-client.js: устранена гонка, из-за которой HTTP-транспорт никогда не отправлял первый claim. onOpen вызывался из микротаски — раньше присваивания wsRef.current, поэтому пейринг-запрос тихо терялся и UI вечно висел на «creating…». Колбэк перенесён в макротаску (setTimeout(…,0)), к этому моменту хэндл уже присвоен http-client.js: fixed a race where the HTTP transport never sent the initial claim. onOpen fired from a microtask before wsRef.current was assigned, so the pairing request was silently dropped and the UI hung on "creating…" forever. The callback now runs as a macrotask (setTimeout(…,0)), by which time the handle is assigned

ВложенияAttachments

  • transport.js: картинки в чате снова грузятся. Relay-хэндл не пробрасывал методы uploadBlob/downloadBlob наружу (они оставались только на внутреннем клиенте), поэтому отправка/приём изображений падали с no-client и вложение не появлялось. Методы теперь проброшены на хэндл transport.js: chat images load again. The relay handle didn't forward the uploadBlob/downloadBlob methods (they lived only on the inner client), so sending/receiving images failed with no-client and the attachment never appeared. The methods are now forwarded onto the handle
