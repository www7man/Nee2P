v0.8.3 — Open-source инфраструктура + UX polish

v0.8.3 май 2026 opensource звонки UX

Docker, CI, и шлифовка вокруг клавиатуры и звонков

  • Docker compose: docker compose up --build — относительно npm install это шаг вперёд для self-hosters. Образ на node:20-alpine, non-root, healthcheck
  • GitHub Actions CI: smoke wire-format на Node 18/20/22, Docker-build, npm audit — больше не нужно гадать «а собирается ли это вообще». Зелёный значок на репо.
  • SECURITY.md / CONTRIBUTING.md / issue + PR templates: репозиторий впервые выглядит как настоящий open-source проект, а не дамп файлов
  • README пересобран: tagline, бэйджи, сравнительная таблица с Session / SimpleX / Briar, секция Docker, ссылки на политику безопасности
  • Pre-flight network check: перед каждым звонком запускается NeeCall.diagnose() — если что-то сломано, видишь «отказано в микрофоне / NAT блокирует / нет интернета» сразу, вместо 45-секундной тишины
  • Keyboard UX: убран translateY-хак, который сжирал первый тап по input на iOS Safari; CreatedScreen больше не autoFocus-ит input — клавиатура не выскакивает сразу после создания сессии
02 — История версий

Полная история

v0.8.2 май 2026 звонки UX

Network diagnostics + мягкое offline-гейтирование

  • NeeCall.diagnose(): проверяет WebRTC, HTTPS, микрофон, STUN-доступность через 2 независимых провайдера, определяет тип NAT (cone vs symmetric)
  • Кнопка «Проверить сеть» в overlay при неудачном звонке — модал с чеклистом и конкретными рекомендациями («попробуйте мобильные данные», «разрешите доступ в настройках»)
  • STUN-пул расширен до 6 серверов: Google, Cloudflare, Twilio, Nextcloud + endpoint на :443 для сетей, пропускающих только HTTPS-порт
  • Кнопка звонка больше не блокируется жёстко при офлайн-пире — SSE-статус устаревает за ~30s; кнопка диммируется с подсказкой «попробовать»
  • Overlay при ошибке авто-скрывается через 10s (было 5s) — пользователь успевает прочитать ошибку
v0.8.1 май 2026 звонки UX

Типизированные ошибки звонков + UX polish

  • Typed error codes: mic-denied, mic-error, ice-failed, peer-busy, peer-rejected, peer-missed, timeout — вместо единого «connection-failed»
  • Кнопка звонка серая и заблокирована когда пир офлайн — исключает 45s-ожидание в никуда
  • Тост в чате при peer-исходах: «Собеседник отклонил звонок», «Линия занята», «Не ответил» — виден ~3.5s
  • Overlay показывает конкретную причину: «Микрофон недоступен», «Сеть блокирует звонки» и др.
  • Overlay авто-скрывается через 5s с обратным отсчётом — не нужно нажимать «Завершить» вручную
  • Исходящий звонок авто-отменяется через 45s с «Не ответил»
  • Fix: сигналы звонка буферизуются если sender-key ещё не готов — устранён race condition
v0.8.0 май 2026 звонки WebRTC

WebRTC peer-to-peer аудиозвонки

  • E2E-зашифрованные голосовые звонки для 2-party сессий
  • Медиа-канал защищён DTLS-SRTP нативно — ключи не покидают браузер
  • Сигналинг (SDP + ICE) передаётся через существующий чат-канал как новый wire type signal — в историю не пишется
  • MVP: STUN-only без TURN; при неудачном NAT traversal — «Нет соединения» с подсказкой
  • UI: кнопка звонка в заголовке чата (появляется когда пир подключён), bottom-sheet для входящего, overlay с таймером + mute / speaker / hangup
  • +812 строк кода: webrtc.js (360), nee2p-screens.jsx (220), nee2p-app.jsx (201), server.js (26)

v0.7.0 май 2026 UX

3-шаговый визард создания сессии

ШагЭкранЧто объясняется
1 Секретная фраза Обрабатывается в браузере через Argon2id; на сервер уходит только MD5-хеш — восстановить фразу невозможно
2 Параметры сессии Срок жизни (1 ч — 7 дней) и число участников (2–8) задаются один раз
3 Пароль входа Только на устройстве, не передаётся по сети, не участвует в шифровании сообщений
  • Индикатор прогресса — анимированные пилюли между экранами
  • Кнопка назад возвращает на предыдущий шаг
  • Кнопка «Далее» без фразы → автоматически генерирует безопасный код

v0.5.0 май 2026 PWA офлайн

PWA: установка и офлайн-режим

  • Service Worker (sw.js): network-first для навигации, cache-first для ассетов
  • manifest.json: приложение устанавливается на домашний экран
  • Web Push уведомления (push.js) — опционально, с явным разрешением
  • IndexedDB persistence (persistence.js): сессия переживает перезагрузку страницы с опцией «Запомнить на этом устройстве»
  • Иконки 192×192 и 512×512
