Создай статическую HTML-страницу `trust.html` в корне проекта hush.

### 1. Главное: сервер — слепой

**Почему это ключевое:**  
Самая распространённая угроза — «сервис читает ваши сообщения». В hush. сервер  
физически не может этого сделать, даже если захочет или будет взломан.

**Что происходит:**
- Ваша фраза → через Argon2id/PBKDF2 → часть ключа (не покидает браузер)
- Ваш браузер генерирует одноразовую X25519-пару (новая при каждом подключении)
- Браузеры обмениваются публичными ключами через сервер (тот видит только публичные)
- ECDH + KEM + phrase → итоговый AES-256-GCM ключ (не покидает браузер)
- Сообщение шифруется: `AES-256-GCM(key, iv_12bytes, plaintext)` → сервер получает `{iv, ct}`
- Сервер пересылает `{iv, ct}` второму участнику. Он не знает ни `iv_plaintext`, ни содержимого

**Что сервер реально видит:**
```
{
  iv:  "<12 случайных байт в base64>",
  ct:  "<зашифрованный текст в base64>"
}
```
Это всё. Без ключа — бессмысленный набор байт.

**Как проверить:**  
Открой DevTools → Network → WS или /r/send.  
Все исходящие пакеты содержат только `iv` и `ct`. Никакого plaintext.

### 2. Криптографический стек (технические детали)

Форматируй как таблицу или список «что / параметры»:

| Компонент | Алгоритм | Параметры |
|-----------|----------|-----------|
| KDF (основной) | Argon2id | time=3, mem=64 МиБ, parallelism=1, output=32 байта |
| KDF (fallback) | PBKDF2-SHA256 | 600 000 итераций, output=32 байта |
| KDF соль | domain-separated | `hush.v3.argon2id:<roomId>` |
| Key Exchange | X25519 (ECDH) | эфемерная пара, уничтожается после сессии |
| Post-quantum | ML-KEM-768 (FIPS 203) | pubkey 1184 байта, ct 1088 байт, shared 32 байта |
| Финальный ключ | HKDF-SHA256 | mix: ECDH shared ‖ phrase key ‖ KEM shared |
| Шифрование | AES-256-GCM | IV 12 байт (случайный, уникальный на сообщение) |
| Fingerprint | SHA256(sorted pubkeys) → BIP-39 | 12 слов = 132 бита энтропии |

Заметка: «Если взломан X25519 — ML-KEM-768 защищает. Если взломан ML-KEM — защищает X25519. Оба должны быть взломаны одновременно.»

---

### 3. Safety Fingerprint — проверка человеком

Объяснить: после установки соединения оба участника видят одни и те же **12 слов BIP-39**.  
Это отпечаток ваших публичных ключей (SHA-256 от конкатенации, отсортированной по значению).

Если слова совпали через другой канал (голос, другой мессенджер) — между вами нет посредника.  
Если слова не совпали — кто-то подменяет ключи (MITM).

Найти: в интерфейсе чата → «Безопасность сессии» → показывает 12 слов и полный hex SHA-256.

### 4. Время жизни данных

- Сообщения хранятся **только в RAM сервера**, пока сессия активна
- Поддерживаемый TTL: 1 час, 6 часов, 24 часа, 3 дня, 7 дней (выбирается при создании)
- При истечении TTL или когда все участники закрыли вкладку → комната удаляется, данные исчезают
- Никакой базы данных, никакого диска, никаких бэкапов
- Файлы и голосовые: хранятся в RAM как зашифрованный blob (до 5 МБ на файл), удаляются вместе с комнатой
- Никаких логов сообщений на сервере

Как проверить: исходный код `server.js` — `class Room` использует `this.history = []` (обычный массив в памяти).

### 8. Что нельзя проверить без дополнительных инструментов

Честный раздел — покажи, что мы не скрываем ограничения:

- **Соответствие кода на продакшне исходнику:** ты видишь код на GitHub, но не можешь  
  автоматически проверить, что именно этот код крутится на сервере.  
  Решение: запусти сам (см. выше) или проверь DevTools что JS-файлы не изменены относительно GitHub.

- **Память сервера после краша:** мы говорим что не логируем — но ты не можешь  
  это проверить снаружи. Доверяй RAM-only архитектуре или запусти свой сервер.

- **Компрометация браузера:** если у тебя установлено вредоносное расширение или  
  ОС под контролем — никакая крипто не поможет.

### 9. FAQ (коротко)

**В: Вы видите IP-адреса пользователей?**  
О: Технически — да, как любой сервер. IP не связан с содержимым переписки. Для полной анонимности используй Tor.

**В: Фраза улетает на сервер?**  
О: Нет. Фраза обрабатывается только в твоём браузере через Argon2id → становится частью ключа шифрования. На сервер уходит только `room = md5(phrase)` — идентификатор комнаты.

**В: Что если сервер сломается во время переписки?**  
О: Сообщения теряются (они только в RAM). Это сделано намеренно — никакого постоянного хранилища.

**В: Можно ли это прослушать на уровне сети?**  
О: Трафик идёт через HTTPS (TLS). Даже если перехватить — видны только зашифрованные пакеты `{iv, ct}`, которые без AES-ключа расшифровать нельзя.
